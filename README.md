# 말씀정원

13명이 2026년 7월 27일부터 8월 9일까지 함께 사용하는 2주 모바일 큐티 웹앱입니다. 단체 채팅방에는 Sites의 공용 주소 하나만 공유하고, 참여자는 첫 방문에 자기 이름을 선택합니다. 13명 모두가 두 주의 현황을 보고 서로의 읽음 기록을 수정할 수 있으며, 매주 누적 완료 수는 별도의 아홉 단계 공동 말씀나무로 표현됩니다. 각 주 정원은 56회에 완성되고 57–91번째 참여도 완성 정원에 계속 기록됩니다.

배포된 사이트와 D1은 클라우드에서 동작하므로 운영자의 컴퓨터를 꺼도 참여자는 계속 사용할 수 있습니다. `npm run dev`로 연 로컬 개발 주소만 해당 컴퓨터가 켜져 있을 때 사용할 수 있습니다.

## 콘텐츠 원칙

사이트 안에서 읽는 기본 본문은 『성경전서 개역한글판』이고, 해설은 본문을 바탕으로 새로 쓴 `말씀정원 자체 해설`입니다.

- 대한성서공회의 안내에 따라 저작재산권 보호기간이 지난 개역한글판을 저작자와 역본명을 표시하고 원문을 변경하지 않는 조건으로 내장합니다.
- 14일 본문에서 한 절을 선택해 복사하거나 모바일 공유창으로 카카오톡 등에 보낼 수 있습니다.
- 개역개정·새번역·쉬운성경과 매일성경·매일성경 순은 권리자의 별도 허가 전까지 전문을 저장하지 않고 공식 페이지로만 연결합니다.
- 말씀정원 해설은 성서유니온 해설을 복사하거나 바꾸어 쓴 글이 아니며, 공식 매일성경 해설과 화면에서 명확히 구분합니다.
- 외부 페이지를 크롤링·프록시하거나 iframe으로 삽입하지 않습니다.
- 출판사와 권리자로부터 표시·복사 범위가 명시된 허가 자료를 받으면 관리자 가져오기로 추가 `embedded` 판본을 활성화할 수 있습니다.

## 로컬 실행

Node.js 22.13 이상이 필요합니다.

```bash
npm install
npm run dev
```

기본 주소는 `http://localhost:3000`입니다. D1은 로컬 Cloudflare 런타임에 연결되고 첫 API 요청에서 스키마와 14일 캠페인 데이터를 멱등하게 준비합니다.

로컬 실행에는 Git에서 제외된 `.dev.vars`의 두 비밀값이 필요합니다.

- `WORD_GARDEN_SEED_JSON`: 13명의 이름, 이전 초대 방식과 호환하기 위한 참여자 토큰 해시, 관리자 토큰 해시
- `WORD_GARDEN_PURGE_SECRET`: 보관 종료일 삭제 작업 전용 Bearer 비밀값

`outputs/invite-tokens.json`에는 이전 참여자 초대 방식과 관리자 접속에 쓰는 원문 토큰이, `outputs/site-secrets.json`에는 Sites에 비밀 환경 변수로 등록할 값이 있습니다. 공용 참여자 링크 운영에는 참여자 원문 토큰이 필요하지 않습니다. 이 파일들과 `.dev.vars`, `.wrangler/`는 `.gitignore` 대상이지만, 현재 작업 폴더가 OneDrive 아래에 있으므로 Git 제외가 클라우드 동기화까지 막아 주지는 않습니다. 실제 운영값은 암호화된 비밀 저장소에 옮기고 화면 공유·로그·이슈·채팅·커밋에 노출하지 마세요.

## 공용 링크와 배포

단체 채팅방에는 최종 Sites 루트 주소 하나만 공유합니다. 참여자는 같은 주소로 접속해 첫 화면에서 자기 이름을 고르고, 동의한 뒤 오늘 말씀과 함께 화면을 사용합니다. 선택한 이름은 해당 브라우저의 쿠키에 기억되며 이름을 바꿔 다시 선택할 수 있습니다.

이 방식은 모임 편의를 우선한 신뢰 기반 운영입니다. 공용 주소를 아는 사람은 이름 목록을 보고 어느 이름이든 선택할 수 있으며 13명의 기록을 조회·수정할 수 있으므로 주소는 모임 채팅방 안에서만 공유합니다. 관리자 링크는 공용 주소와 별도로 보관하고 단체방에 올리지 않습니다. 관리자 자격 증명은 별도 API로 회전할 수 있으며, 새 관리자 링크는 응답에서 단 한 번만 확인할 수 있습니다.

`/invite/:token` 경로는 이전 참여자 링크와 관리자 링크 호환을 위해 남아 있을 수 있지만, 일반 참여자에게는 개인 링크를 발급하거나 배포하지 않습니다.

Sites 배포는 먼저 소유자만 접근할 수 있는 비공개 미리보기로 검수합니다. 공개 또는 링크 공유 상태로 바꾸는 작업은 운영자의 명시적 승인 후에만 수행하고, 승인 전에는 참여자에게 주소를 보내지 않습니다. `noindex`와 `robots.txt`는 검색 노출 억제 장치일 뿐 접근 제어가 아닙니다.

## 검증

```bash
npm run check
npm run test:security:live
```

`npm run check`는 린트, 타입 검사, 단위·통합 테스트와 프로덕션 빌드를 실행합니다. 라이브 보안 검사는 로컬 비밀 파일이 있는 개발 환경에서만 실행하며, 출력이나 접근 로그에 관리자 원문 주소나 비밀값이 남지 않도록 결과를 외부에 공유하지 않습니다.

## 데이터 수명

- 2026-07-26까지: 안내 화면, 체크 잠금
- 2026-07-27–08-02: 1주차 정원 운영
- 2026-08-03–09: 2주차 정원을 0부터 운영하며 1주차 보충 체크도 허용
- 2026-08-10부터: 두 정원 읽기 전용
- 2026-09-09 00:00 KST부터: 실명, 초대, 세션, 개인 완료, 감사 기록 삭제 가능

삭제는 첫 사용자 요청에서도 지연 실행되지만, 정시 처리를 위해 `POST /api/maintenance/purge`를 외부 예약 작업으로 반드시 한 번 이상 호출해야 합니다. 삭제 후 익명 공동 완료 합계만 남습니다. D1 Time Travel에는 삭제 전 상태가 Free 요금제에서 최대 7일, Paid 요금제에서 최대 30일 복구 가능할 수 있으므로, 행 삭제를 즉시 물리적 소거로 간주하면 안 됩니다.

배포·공용 링크 안내·관리자 자격 증명 회전·예약 삭제·D1 백업까지 포함한 절차는 [운영 런북](docs/operations.md)을 따릅니다.

## 주요 경로

- `/`: 단체방 공용 진입 주소와 첫 방문 이름 선택
- `POST /api/session`: 선택한 참여자 이름을 현재 브라우저에 설정
- `DELETE /api/session`: 현재 이름 선택을 지우고 다시 선택
- `/invite/:token`: 이전 초대 방식과 관리자 접속을 위한 호환 경로
- `/day/:date`: 날짜별 말씀과 내 체크
- `/together`: 주차별 13명 × 7일 현황과 참여자 간 공동 체크 수정
- `/admin`: 읽음 정정, 공용 주소 안내, 허가 콘텐츠와 관리자 기능 관리
- `PUT /api/completions/shared/:participantId/:dayId`: 이름을 선택한 참여자가 지정한 사람의 기록 정정
- `POST /api/admin/credentials/rotate`: 관리자 자격 증명 회전
- `POST /api/maintenance/purge`: 보관 종료일 개인정보 삭제

검색엔진은 메타데이터와 `public/robots.txt` 양쪽에서 차단합니다. 광고, 공개 분석 도구, 카카오 로그인, 자동 크롤링은 사용하지 않습니다.
