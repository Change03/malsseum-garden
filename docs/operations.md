# 말씀정원 운영 런북

이 문서는 2026년 7월 27일–8월 9일의 2주 운영과 9월 9일 개인정보 삭제를 위한 체크리스트입니다. 모든 시각 판단은 `Asia/Seoul`을 기준으로 합니다.

## 1. 비밀값과 로컬 파일

운영에 사용하는 비밀값은 다음 두 개입니다.

| 환경 변수 | 용도 | 취급 방법 |
|---|---|---|
| `WORD_GARDEN_SEED_JSON` | 13명 이름, 이전 초대 방식 호환용 참여자 토큰 해시, 관리자 토큰 해시로 최초 D1 데이터 구성 | Sites와 로컬 런타임에서 secret으로만 등록 |
| `WORD_GARDEN_PURGE_SECRET` | 보관 종료일 삭제 API의 Bearer 인증 | 앱 운영 비밀과 분리해 예약 작업에 최소 권한으로 등록 |

로컬 민감 파일은 다음과 같습니다.

| 파일 또는 폴더 | 포함 정보 | 운영 원칙 |
|---|---|---|
| `.dev.vars` | 두 환경 변수의 실제 값 | 개발 장비에서만 사용 |
| `outputs/invite-tokens.json` | 이전 참여자 초대 방식과 관리자 접속에 쓰는 원문 토큰 | 공용 참여자 링크 운영에는 사용하지 않으며 암호화 보관 또는 안전 삭제 |
| `outputs/site-secrets.json` | Sites 환경 변수에 올릴 실제 값 | 등록 확인 후 안전 삭제 |
| `.wrangler/` | 로컬 D1과 런타임 로그 | 테스트 종료 뒤 개인 데이터와 토큰 로그가 없는지 확인 후 삭제 |

이 경로들은 Git에서 제외되어 있지만 `.gitignore`는 OneDrive, 백업 프로그램, 백신 격리 저장소의 복제를 막지 않습니다. 이 프로젝트처럼 OneDrive 아래에서 작업할 때는 운영 비밀 파일을 동기화 제외 위치의 암호화 저장소로 옮기거나 OneDrive 동기화 범위를 별도로 제한합니다.

배포 전 다음 명령으로 민감 파일이 추적되지 않는지 확인합니다.

```bash
git status --short
git check-ignore -v .dev.vars outputs/invite-tokens.json outputs/site-secrets.json .wrangler
```

비밀값 자체를 터미널에 출력하는 명령, 접근 토큰이 포함된 URL을 `curl -v`로 호출하는 명령, 관리자 URL을 CI 로그에 남기는 검사는 운영 토큰으로 실행하지 않습니다.

## 2. 비공개 미리보기와 공개 승인

1. Sites에 `WORD_GARDEN_SEED_JSON`과 `WORD_GARDEN_PURGE_SECRET`을 일반 변수가 아닌 secret으로 등록합니다.
2. 소유자 전용 비공개 버전을 배포합니다.
3. 모바일 화면, 첫 방문 이름 선택, 참여자 간 공동 기록 수정, 관리자 전용 기능 분리, KST 잠금, 외부 링크 모드를 검수합니다.
4. 운영자가 공개 배포를 명시적으로 승인했는지 기록합니다.
5. 승인 후에만 Sites 접근 상태를 공개 또는 링크 공유로 변경합니다.
6. 최종 공개 루트 주소가 확정되면 그 주소 하나만 단체 채팅방에 공유할 문구로 준비합니다.

비공개 미리보기 주소는 참여자가 사용할 수 없습니다. 반대로 공개 주소에서는 첫 화면의 이름 선택 목록이 의도적으로 보입니다. 이름을 선택하지 않은 상태의 `/api/week`가 `401`이고 주간 완료 데이터가 오류 응답에 포함되지 않는지 확인합니다. `noindex`와 `robots.txt`는 접근 제어가 아닙니다.

## 3. 공용 링크 배포와 이름 선택

참여자에게는 `https://최종주소/` 형식의 루트 주소 하나만 카카오톡 단체방에 보냅니다.

- 첫 방문자는 자기 이름을 선택하고 공개 범위에 동의한 뒤 사용합니다.
- 선택한 이름은 브라우저 쿠키에 기억됩니다. 한 기기를 함께 쓰거나 잘못 골랐다면 이름 바꾸기로 선택을 지운 뒤 다시 고릅니다.
- 13명 모두가 같은 주소에서 전체 현황을 보고 누구의 읽음 기록이든 수정할 수 있습니다.
- 공용 주소를 아는 사람은 어느 이름이든 선택하고 기록을 볼 수 있으므로 모임 밖으로 전달하지 않습니다.
- 관리자 링크는 공용 주소와 분리해 암호화된 비밀 저장소에 보관하고 단체방에 올리지 않습니다.
- `/invite/:token`은 이전 참여자 링크와 관리자 접속 호환을 위해 남아 있을 수 있지만 일반 참여자용 링크로 새로 만들거나 배포하지 않습니다.

배포된 Sites 앱과 D1은 클라우드에서 실행되므로 운영자의 컴퓨터를 꺼도 사이트는 계속 동작합니다. 개발자가 `npm run dev`로 연 `localhost` 주소만 개발 컴퓨터가 꺼지면 접속할 수 없습니다.

## 4. 관리자 자격 증명 회전

관리자 링크가 노출됐거나 담당자가 바뀌면 `POST /api/admin/credentials/rotate`를 호출합니다. 이 요청은 현재 관리자 세션과 같은 출처의 `Origin`이 있어야 하므로, 로그인된 `/admin` 탭에서 같은 출처 요청으로 실행합니다.

브라우저 개발자 도구의 Console에서 다음 요청을 실행할 수 있습니다.

```js
await fetch("/api/admin/credentials/rotate", { method: "POST" }).then((response) => response.json())
```

회전 전에 암호화된 저장 위치를 준비합니다. 성공 응답의 `credential.inviteUrl`과 `credential.token`은 단 한 번만 표시되고, 성공 즉시 기존 관리자 세션 전체가 폐기됩니다. 응답을 잃으면 일반 화면에서 새 토큰을 다시 조회할 수 없으므로 결과를 확인한 뒤 Console 기록과 클립보드를 지웁니다. 새 관리자 링크로 다시 접속해 `/admin`을 열 수 있는지 확인하고, 이전 링크가 거부되는지도 확인합니다.

## 5. 저작권 안전 모드

허가 전 기본 데이터의 모든 콘텐츠 변형은 `external` 모드입니다.

- 서버에는 날짜, 본문 범위, 역본·해설판 이름, 공식 출처 URL만 둡니다.
- 성서유니온 내부 Ajax 주소를 호출하거나 공식 페이지를 크롤링·프록시·iframe 삽입하지 않습니다.
- 해설은 복사 대상으로 제공하지 않습니다.
- 사용자가 입력한 한 절은 브라우저 메모리에서만 공유 문구를 만들고 서버·D1·브라우저 저장소에 보관하지 않습니다.

`embedded` 모드는 서면 허가 자료에 모바일 웹 전문 표시, 대상 역본·해설판, 기간, 한 절 공유 여부와 최대 분량이 모두 명시된 경우에만 사용합니다. 권리 만료일과 출처를 콘텐츠 가져오기 데이터에 넣고, 허가되지 않은 조합은 계속 비활성화합니다.

- 허가받지 않은 본문과 해설 전문이 응답에 포함되지 않는지

## 7. 2026년 9월 9일 예약 삭제

앱은 보관 종료 뒤 첫 요청에서도 삭제를 시도하지만 방문이 없을 수 있으므로 외부 예약 작업이 필요합니다. 2026년 9월 9일 00:05 KST, 즉 2026년 9월 8일 15:05 UTC 이후에 다음 요청을 실행하고 실패 시 재시도합니다.

```bash
curl --fail-with-body --request POST \
  --header "Authorization: Bearer ${WORD_GARDEN_PURGE_SECRET}" \
  "https://최종주소/api/maintenance/purge"
```

예약 작업 설정에는 `WORD_GARDEN_PURGE_SECRET` 값을 직접 스크립트에 쓰지 말고 해당 서비스의 secret 저장 기능을 사용합니다.

응답 기준은 다음과 같습니다.

| HTTP | 응답 상태 | 의미 |
|---:|---|---|
| 200 | `purged` | 익명 총합을 남기고 개인정보 삭제 완료 |
| 200 | `already_purged` | 이미 삭제되어 추가 작업 없음 |
| 423 | `RETENTION_NOT_DUE` | KST 보관 종료 전 호출 |
| 401 | `INVALID_PURGE_TOKEN` | Bearer 비밀값 불일치 |

호출은 멱등이므로 네트워크 응답을 받지 못했다면 같은 요청을 재시도할 수 있습니다. 완료 후 D1에서 다음 상태를 확인합니다.

- `campaigns.status = 'deleted'`
- `campaigns.personal_data_deleted_at`이 비어 있지 않음
- `campaigns.anonymous_total`에 공동 완료 합계가 남음
- `participants`, `sessions`, `completions`, `audit_logs`, `admin_credentials`의 해당 캠페인 행이 0개

## 8. 삭제 뒤 비밀값과 서비스 정리

현재 구현은 콜드 스타트에서 `WORD_GARDEN_SEED_JSON`을 읽어 초기화를 검증합니다. 따라서 삭제 API 성공 직후 사이트를 계속 제공하면서 시드 비밀값만 먼저 제거하면 새 런타임 요청이 실패할 수 있습니다. 다음 순서를 지킵니다.

1. 삭제 결과와 익명 합계를 확인합니다.
2. Sites를 소유자 전용으로 되돌리거나 운영 종료 버전으로 교체해 일반 접근을 차단합니다.
3. 예약 작업에서 `WORD_GARDEN_PURGE_SECRET`을 제거합니다.
4. Sites 환경에서 `WORD_GARDEN_PURGE_SECRET`과 `WORD_GARDEN_SEED_JSON`을 제거합니다.
5. 로컬 `.dev.vars`, `outputs/invite-tokens.json`, `outputs/site-secrets.json`, 운영 토큰이 남은 로그와 클립보드 기록을 안전 삭제합니다.
6. 로컬 `.wrangler/`에 실명이나 완료 기록이 있다면 안전 삭제합니다.

익명 합계를 계속 온라인에서 보여 주려면 시드 비밀값 제거 전에 개인정보를 다시 시드하지 않는 별도 종료 버전을 배포해야 합니다. 현재 버전을 그대로 둔 채 환경 변수만 삭제하는 방식은 지원되지 않습니다.

## 9. D1 Time Travel과 백업 주의사항

D1의 production 저장소는 Time Travel이 항상 켜져 있고, 삭제 전 시점으로 복원할 수 있습니다. 2026년 4월 21일 기준 공식 제한은 Workers Free 요금제 최대 7일, Paid 요금제 최대 30일입니다. 따라서 9월 9일의 SQL 행 삭제는 활성 데이터에서는 즉시 반영되지만, 삭제 전 개인정보가 같은 순간 모든 복구 기록에서 물리적으로 사라졌다는 뜻은 아닙니다.

- 삭제 이후 과거 시점으로 Time Travel 복원을 실행하지 않습니다.
- 실제 삭제 시각과 당시 요금제를 운영 기록에 남깁니다.
- Free라면 삭제 시각으로부터 최소 7일, Paid라면 최소 30일이 지난 뒤 복구 가능 기간이 끝났는지 다시 확인합니다.
- D1 데이터베이스 자체를 삭제하면 애플리케이션이 동작하지 않습니다. 전체 DB 삭제가 필요한 정책이라면 먼저 사이트를 종료하고 익명 합계 유지 여부를 결정합니다.
- Cloudflare 공개 문서만으로 삭제된 데이터베이스의 모든 내부 복제본 소거 시점을 단정할 수 없습니다. 즉시·검증 가능한 물리적 삭제가 계약 요건이면 Cloudflare에 보존·삭제 정책을 별도로 확인한 뒤 운영합니다.

- [Cloudflare D1 limits](https://developers.cloudflare.com/d1/platform/limits/)
- [Cloudflare D1 데이터베이스 삭제 안내](https://developers.cloudflare.com/d1/get-started/#7-optional-delete-your-database)

## 10. 최종 종료 기록

운영자는 비밀값 자체를 적지 않고 다음 항목만 남깁니다.

- 공개 승인자와 승인 시각
- 행사 시작·종료 시각
- 삭제 API 성공 시각과 `purged` 또는 `already_purged` 상태
- 삭제 직후 테이블 검증 결과
- 사용한 D1 요금제와 Time Travel 만료 확인 예정일
- Sites 접근 차단, 환경 비밀 삭제, 로컬 민감 파일 삭제 완료 시각
